oitg.results API¶

oitg.results.load_hdf5_file(filename)[source]¶

Load an ARTIQ results file.

Returns

A dictionary containing the logical contents of the HDF5 file, including:

  • "start_time": the unix timestamp when the experiment was built

  • "expid": experiment description, including submission arguments

  • "datasets": dictionary containing all set datasets

oitg.results.load_result(day=None, rid=None, experiment=None, root_path=None)[source]¶

Find and load an HDF5 results file from an ARTIQ master results directory.

The results file is described by a rid and a day (provided date string, defaults to today). See find_results() for a full description of the arguments.

Returns

A dictionary containing the contents of the file; see load_hdf5_file().

oitg.results.find_results(day=None, rid=None, hour=None, class_name=None, experiment=None, root_path=None)[source]¶

Find the ARTIQ result files for a given experiment.

The ARTIQ results root search path is either the standard root path for the given experiment name (see oitg.paths.artiq_results_path()) or root_path (if root_path is given).

Parameters

  • rid – An experiment run ID or list of run IDs to load. If None loads all rids.

  • hour – hour or list of hours when the experiment was build. If None loads all hours.

  • day – Datestring or list of datestrings. Defaults to today. Dates are in ISO format (yyyy-mm-dd).

  • class_name – is the experiment class name, or list of names. If None loads all classes.

Returns

A dict of results, indexed by rid. The dict entries are a named tuple (path, day, hour, cls).
