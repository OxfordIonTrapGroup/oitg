oitg.threshold API¶

oitg.threshold.calc_target_bin_time(bright_rate, dark_rate, p_error_target, p_bright=0.5)[source]¶

Calculate optimal threshold bin time for target error chance

The calculation assumes both bright and dark counts are Poisson distributed and gives a threshold minimising the error probability

The calculation neglects de-shelving and accidental shelving during the readout bin time. It is therefore not suitable for P(error) < 2e-4. See thesis: Alice Burrell, 2010

Parameters

  • bright_rate – expected bright count rate in $s^-1$

  • dark_rate – expected dark count ratein $s^-1$

  • p_error_target – target error probability

  • p_bright – probability of encountering a bright state (default=0.5)

Returns

(target_t_bin [s], threshold_rate [$s^-1$])

oitg.threshold.calc_thresh_rate(bright_rate, dark_rate, t_bin=0.001, p_bright=0.5)[source]¶

Optimal threshold rate for distinguishing bright and dark states

The calculation assumes both bright and dark counts are Poisson distributed and gives a threshold minimising the error probability.

The calculation neglects de-shelving and accidental shelving during the readout bin time. It is therefore not suitable for P(error) < 2e-4. See thesis: Alice Burrell, 2010

Parameters

  • bright_rate – expected bright count rate in $s^-1$

  • dark_rate – expected dark count ratein $s^-1$

  • t_bin – integration time in s. Only relevant for p_bright!=0.5 (default=1e-3)

  • p_bright – probability of encountering a bright state (default=0.5)

Returns

threshold_rate /$s^-1$
